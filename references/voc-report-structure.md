# 14-模块VOC品类报告结构

> 参考风格：wenmai-ai.com/reports/juicer-top10-category-report/ 羊皮纸暖色报告（NOT 暗色）
> 适用场景：3~10个竞品ASIN的VOC分析，生成完整的品类洞察HTML报告

## 14大模块（按顺序，缺一不可）

### 0. 一页决策卡（v5+要求）
- 表格形式展示：研究范围/竞争格局/品类均价/平均评分/核心话题/创新缺口/入局风险
- 7行结论，快速决策

### 1. 数据说明与质量分级（v5+要求）
- 表格：数据块 x 可用性 x 可信级别 x 处理方式

### 2. 品类总览 Dashboard
- 顶部：4个KPI卡片（覆盖评论数/均价/平均评分/正负比率）
- 价格柱状图 + 情感分布饼图
- 产品一览表（品牌/ASIN/价格/评分/评论量/月销/一句话点评）
- **不要**额外加独立的月销KPI行（10列小框框）——用户要求删除

### 3. 品类总体分析总结 - AI Summary（v5+要求）
- 羊皮纸风格卡片（ai-card），4段分析
- 竞争格局 -> 品类健康度 -> 用户情感基调 -> 入局建议

### 4. PSPS 用户画像 (品类级)
三列布局：Persona x Scenario x Pain，各5项+%

### 5. ABSA 方面级情感分析
- TOP8话题柱状图 + TOP4产品雷达
- 表格：话题 x 提及率 x 方向 x ACT建议

### 6. APPEALS 8维竞争力分析
- 雷达图：10款ASIN x 8维度（功能效果/易用性/安全性/品质耐久/性价比/便携设计/品牌信任/售后支持）

### 7. KANO 需求分类
- 三栏卡片：基本型/期望型/魅力型，带进度条+百分比+描述

### 8. 痛爽痒三维图谱
- 三列布局（pain/joy/itch-col），每列5项，带进度条+提及率%

### 9. 用户旅程摩擦分析
- 8阶段雷达图(1-10分) + 每阶段详解卡片 + 场景描述+改进建议

### 10. 逐品 ASIN 拆解
- 10个 `<details>` 折叠卡，每个内含：
  - 5列KPI（价格/评分/评论量/月销/抓取数）
  - 环形情感饼图 (ECharts radius:["40%","65%"])
  - 话题分布柱状图
  - 好评精选5条（含中文简译）
  - 差评精选5条（含中文简译）
  - **产品创新机会**（必须！）：3条升级方向用① ② ③列出

### 11. 品类好差评横向对比 + 各产品定位
**不要用堆叠柱状图！** 参考样式：
- **好评核心维度表**：4列（维度 / 覆盖ASIN / 进度条 / 代表好评原文）
- **差评核心痛点表**：4列（同理）
- **各产品整体定位评价**：2列网格卡片，每张含emoji+品牌+价格+一句话定位

### 12. 品类创新机会汇总
- 立刻能做(3条) + 短期突破(3条) + 长期布局(3条)
- 入局策略(4条) + 量化投入产出评估表

### 13. 侧边栏导航
- 右侧固定悬浮导航杆(hover展开)，14个锚点链接

## 关键陷阱

1. **子代理数据污染**：没有API权限时子代理会虚构价格/评分等数据
2. **品牌校验**：B0FCS7NZ35=FEELSO（非FORIOUS）
3. **f-string陷阱**：预计算JSON变量再引用，不在f-string内嵌json.dumps()
4. **中文引号问题**：`"xxx"`（U+201C/U+201D）在Python字符串中被解析为字符串结束，用英文引号或「」
5. **dict[:N]问题**：Python3.9+不适用，用`list(dict.keys())[:N]`
6. **输出路径偏移**：当脚本在子目录（如`data/voc/`）中时，`__file__`的相对`dirname`层级计算容易出错，导致HTML被写入 `data/data/voc/` 而非 `data/voc/`。先`print(__file__)`确认位置再算路径。
7. **旧品类脚本污染**：直发梳脚本的ASINS/PRODUCT/TOPIC_MAP/EN_CN_MAP与厨房水龙头完全不同，强行修改比重写更麻烦。
